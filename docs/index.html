MLX Installation Guide

Prerequisites

Before installing MLX, you need to set up Xcode properly on your Apple Silicon Mac:

1 Install Xcode from the Mac App Store:

  • Open the App Store
  • Search for "Xcode"
  • Click "Get" or "Install"
  • Wait for the download and installation to complete (it's a large download, several GB)

2 After Xcode is installed, open it at least once to accept the initial license agreements

3 Set up the Xcode command line tools with these commands:

$ sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer Password: ******** $ sudo xcodebuild -license accept License accepted

Performance & Benchmarking

System Requirements

  • macOS: 13.5 or later (Sonoma recommended)
  • Python: 3.9 or later
  • Hardware: Apple Silicon (M1/M2/M3)
  • RAM: 8GB minimum (16GB+ recommended)

Key Features

  • Unified Memory: Zero-copy data sharing between CPU and GPU
  • Lazy Evaluation: Operations are only computed when results are needed
  • Metal Integration: Native support for Apple GPU architecture
  • Multi-Device: Seamless CPU/GPU execution switching

Performance Monitoring

$ MTL_CAPTURE_ENABLED=1 python3 your_script.py Enable Metal capture for performance analysis

Optimization Tips

  • Metal Debug Mode: Use MLX_METAL_DEBUG=ON for performance profiling
  • Batch Processing: Optimize batch sizes for your specific M-series chip
  • Memory Management: Monitor unified memory usage patterns
  • Compute Graph: Leverage lazy evaluation for optimal performance

Quick Tips

  • Debug Mode: Build with CMAKE_ARGS="-DMLX_METAL_DEBUG=ON"
  • JIT Compilation: Enable with MLX_METAL_JIT=ON for smaller binaries
  • Memory Usage: Monitor with Activity Monitor's GPU tab
  • Cache Location: Metal shader cache in ~/Library/Caches/com.apple.metal

Common Issues

  • Metal Compiler Missing: Install Xcode Command Line Tools
  • Memory Errors: Reduce batch size or enable LOW_MEMORY=1
  • Slow First Run: Normal due to JIT compilation, subsequent runs will be faster
  • GPU Not Detected: Verify Metal support with system_profiler SPDisplaysDataType

  • Ring Tests: mlx.launch -n 8

Test Components

C++ Tests

Using doctest framework for core functionality testing

TEST_CASE("metal device") {
    CHECK(metal::is_available());
    auto& device = metal::device(Device::gpu);
}

Python Tests

Using unittest for Python API validation

class MLXTestCase(unittest.TestCase):
    def test_array_ops(self):
        x = mx.array([1, 2, 3])
        y = mx.add(x, x)
        self.assertTrue(mx.array_equal(y, mx.array([2, 4, 6])))

Style Guide

Test Results

✓ Success

Exit code: 0

All tests passed successfully

✗ Failure

Exit code: 1

Test failures detected

⚠ Interrupt

Exit code: 130

Test execution interrupted
